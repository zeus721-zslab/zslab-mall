# 액터(Actor) 구조
 
입점형 쇼핑몰의 사용자 계층 구조.
 
## 액터 계층
 
```
                     ┌────────────────────┐
                     │       사용자        │
                     └─────────┬──────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
        ▼                      ▼                      ▼
 
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   구매자     │       │    관리자    │       │ 판매관리자   │
└─────┬───────┘       └─────┬───────┘       └─────┬───────┘
      │                     │                     │
      │                     │                     │
      ▼                     ▼                     ▼
 
┌─────────────┐      ┌─────────────┐
│ 회원등급     │      │ 관리자권한   │
├─────────────┤      ├─────────────┤
│ 실버         │      │ 전체관리자   │
│ 골드         │      │ 운영자       │
│ 플래티넘      │      └─────────────┘
└─────────────┘
```
 
## 액터 정의
 
| 액터 | 설명 |
|------|------|
| 구매자 | 일반 회원. 누적 구매금액에 따른 등급(실버/골드/플래티넘) 자동 부여 |
| 관리자 | 플랫폼 운영 주체. 전체관리자 / 운영자로 분리 |
| 판매관리자 | 입점사 측 관리자. 자사 상품·주문·정산 관리 |
 
## 회원 등급
 
| 등급 | 비고 |
|------|------|
| 실버 (SILVER) | 기본 등급 |
| 골드 (GOLD) | 누적 구매금액 기준 자동 승급 |
| 플래티넘 (PLATINUM) | 최상위 등급 |
 
등급 기준·혜택은 **운영자가 자유롭게 조정 가능**한 정책 테이블로 분리 관리 (별도 설계).
 
## 관리자 권한
 
| 권한 | 설명 |
|------|------|
| 전체관리자 (SUPER_ADMIN) | 모든 권한 (시스템 설정·권한관리 포함) |
| 운영자 (ADMIN_OPERATOR) | 운영 업무 (상품 승인·판매자 관리·정산 조회) |
