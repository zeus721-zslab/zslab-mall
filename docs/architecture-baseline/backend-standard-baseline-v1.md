# 멀티벤더 쇼핑몰 백엔드 기능 표준 기준선 v1

> Phase A 산출물. 특정 코드 무관·일반 멀티벤더 표준 정의. 이후 실제 코드 대조(Phase B)의 기준선.
> 상태명·이벤트명은 표준 일반명이며 실제 구현 enum과 다를 수 있음(B단계 매핑).

## §1. 기능 카탈로그 (15 도메인)

### 1. 회원 (User)
- 가입·프로필·주소록·탈퇴/휴면
- 역할 분리 (구매자·셀러·관리자)

### 2. 인증·인가 (Auth)
- 로그인/토큰
- 권한별 접근제어 (역할별 자원 분리)

### 3. 상품·카탈로그 (Product)
- 등록·수정·옵션/변형·이미지·재고 연동
- 상품 상태 (초안·심사중·판매중·중지)
- 카테고리(taxonomy) 매핑

### 4. 검색 (Search)
- 키워드·카테고리·필터·정렬
- 교차벤더 탐색
- 검색 결과 Pagination (사실상 필수)

### 5. 장바구니 (Cart)
- 담기·수량변경·삭제
- 다벤더 담기 → 주문 분할

### 6. 주문 (Order)
- 생성·조회·취소·상태전이
- 부분취소·부분반품

### 7. 결제 (Payment)
- 승인·취소·부분취소
- 에스크로 (구매확정까지 자금 보류)
- PG 연동

### 8. 배송 (Delivery)
- 송장·배송추적·배송완료
- 이행상태 (미이행·부분이행·이행) — 주문상태와 별개 축

### 9. 클레임/반품·교환 (Claim)
- 취소·반품·교환 접수·승인·거절
- 귀책 판정 (단순변심=구매자, 오배송·불량=셀러)별 배송비 부담
- 선환불 후 검수 (정책 변형)

### 10. 정산 (Settlement)
- 구매확정 → 정산 트리거
- 수수료 차감·순액 지급
- 정산 주기·명세
- 5요소: 캡처·결제·수수료계산·지급·대사(reconciliation)

### 11. 리뷰 (Review)
- 상품평·평점
- verified purchase 기반
- 신고·모더레이션

### 12. 알림 (Notification)
- 주문·배송·정산·클레임 상태 알림 (구매자·셀러 양측)

### 13. 관리자 운영 (Admin)
- 워크플로우형: 셀러승인·상품승인·클레임중재·정산실행·권한관리
- 상시 조회형: 회원·주문·상품·셀러·정산·클레임·감사로그 통합 조회/관리

### 14. 감사 (Audit)
- 운영자 명령·상태변경 추적 로그 (자금 유입·보류·해제·지급 전 과정)

### [선택 기능 — 표준 최소선 제외]
- 구매자 등급: 구매실적 기반 등급·혜택. 마케팅/CRM 성격. 다수 플랫폼이 미보유해도 정상 운영 → 표준 최소선 아닌 선택 기능.

### [Seller Onboarding — §1.2 셀러 도메인 소속]
- 셀러 신원·사업자 검증 (KYC/KYB): 정산받기 전 필수. 인증(Authentication) 아닌 입점(Onboarding) 절차.
- 입점 5단계: 등록·검증·계약(커미션율 약관)·승인·안내

## §2. 3파트 시나리오

### 구매자
가입 → 탐색 → 장바구니 → 주문 → 결제(에스크로) → 배송추적 → 구매확정(정산 트리거) → 클레임(수령 7일내) → 리뷰

### 셀러
입점(KYC/KYB·약관·5단계) → 상품등록(심사) → 재고관리 → 주문수락 → 배송처리 → 정산확인

### 관리자
- 워크플로우: 셀러승인·상품승인·클레임중재·정산실행·권한관리
- 상시 조회: 회원·주문·상품·셀러·정산·클레임·감사로그 통합 조회/관리

## §3. 데이터 흐름

### 3-1. 주문 상태머신
결제대기 →(결제승인)→ 결제완료 →(셀러수락)→ 상품준비 →(출고)→ 배송중 →(배송완료)→ 배송완료 →(구매확정)→ 거래완료
- 결제대기 →(timeout)→ 주문취소
- 결제완료/상품준비 →(취소, 배송 전)→ 주문취소
- 배송완료 →(수령 7일내)→ 클레임(취소·반품·교환)
- (핵심) 구매확정 = 거래완료 확정점. 이후 정산 개시·일반 취소 차단.

### 3-2. 결제 상태머신
결제대기 →(승인)→ 승인완료 →(에스크로 보유)→ 구매확정 시 정산가능
- 결제대기 →(실패)→ 실패
- 승인완료 →(전체취소)→ 환불
- 승인완료 →(부분취소)→ 부분환불

### 3-3. 배송 상태머신
배송준비 →(송장등록)→ 배송중 →→ 배송완료
- 이행상태(미이행·부분이행·이행)는 주문상태와 별개 축

### 3-4. 클레임 상태머신
클레임접수 → 승인심사 →(승인)→ 처리중 → 완료
- 승인심사 →(거절)→ 거절
- 귀책 판정: 단순변심=구매자 배송비 / 오배송·불량=셀러 배송비

### 3-5. 정산 상태머신
정산대기 →(구매확정)→ 정산확정(gross−수수료=순액) →(지급실행)→ 지급완료
- 선택: 클레임·분쟁 시 정산 보류 후 재개 가능 (상태명·구현은 정책. 개념만 표준)

### 3-6. Aggregate 간 의존 방향
> 표준 문서는 "누가 누구에게 영향을 주는가"(의존 방향)만 정의. 동기호출·도메인이벤트·Outbox·MQ 모두 가능한 구현이며 특정 방식을 강제하지 않음.

- 결제승인: Payment → Order 상태 갱신 / Payment → Inventory 재고 확보(차감·예약은 구현정책)
- 출고: Order·Delivery → Order 상태 갱신
- 배송완료: Delivery → Order 상태 갱신
- 구매확정(PurchaseConfirmed): Order → Settlement 정산 개시
- 클레임 승인: Claim → Refund·Payment 환불 / Claim → Inventory 재고 복원(반품 검수 후 조건부)
- 상태변경 전반: 각 도메인 → Notification 알림
- 운영자·상태변경 전반: 각 도메인 → Audit 감사 로그
